# Система онлайн-записи с календарём

Запись клиентов на слоты с **защитой от двойной брони на уровне БД**, гибким
расписанием, корректной работой с таймзонами и напоминаниями в Telegram/email.

Стек: **Next.js 14 (App Router) + TypeScript + Supabase (Postgres) + Tailwind**.
Напоминания — Telegram Bot API / email через Resend. Планировщик — Vercel Cron
(или `pg_cron`).

---

## Что внутри

- **Публичная часть** (`/`) — список услуг, выбор даты в календаре, доступные
  слоты, форма записи, экран подтверждения.
- **Админка** (`/admin`) — управление услугами, рабочими часами, выходными,
  просмотр и отмена ближайших записей.
- **API** — генерация слотов, бронирование, cron отправки напоминаний.

### Ключевые технические решения

| Задача | Решение |
| --- | --- |
| Двойная бронь (race condition) | `EXCLUDE` constraint на пересечение интервалов `tstzrange` через `gist`. БД физически не даёт создать пересекающуюся бронь — конкурентный INSERT падает с ошибкой `23P01`, которую API превращает в `409 Conflict`. Никаких блокировок в коде. |
| Таймзоны | В БД всё в `timestamptz` (UTC). Рабочие часы заданы в `BUSINESS_TIMEZONE`. Пользователю слоты показываются в его браузерной зоне (`Intl` + `date-fns-tz`). |
| Генерация слотов | SQL-функция `get_available_slots(service_id, date, tz)` — единый источник истины. Учитывает рабочие часы, длительность услуги, занятые брони и выходные. |
| Идемпотентность напоминаний | Очередь в таблице `notifications`. RPC `claim_due_notifications` атомарно забирает задачи через `FOR UPDATE SKIP LOCKED`, поэтому одно напоминание уходит ровно один раз даже при параллельных запусках cron. |
| Безопасность | Весь доступ к БД — серверный, под `service_role`. На всех таблицах включён RLS **без политик**, поэтому анонимный доступ закрыт полностью. Браузер с Supabase напрямую не общается. |

---

## Модель данных

- `services` — услуги (название, длительность, цена).
- `working_hours` — шаблон расписания (день недели, начало, конец).
- `time_off` — выходные и исключения (дата, причина).
- `bookings` — записи (услуга, клиент, `starts_at`/`ends_at` в `timestamptz`, статус).
- `notifications` — очередь напоминаний (бронь, канал, когда отправить, статус).

Миграции и сид-данные — в папке [`supabase/`](./supabase).

## Быстрый старт

### 1. Установка

```bash
npm install
```

### 2. Supabase

Создайте проект на [supabase.com](https://supabase.com) и примените миграции.
Откройте в проекте **SQL Editor** и выполните по порядку:

1. `supabase/migrations/0001_schema.sql` — таблицы, индексы, `EXCLUDE` constraint, RLS.
2. `supabase/migrations/0002_functions.sql` — функции слотов и напоминаний, триггеры.
3. `supabase/seed.sql` — тестовые услуги и рабочие часы (по желанию).

> `0003_cron.sql` нужен только если планируете запускать рассылку напрямую из
> Postgres через `pg_cron` вместо Vercel Cron. По умолчанию не требуется.

Если используете Supabase CLI:

```bash
supabase db push        # применит миграции
psql "$DATABASE_URL" -f supabase/seed.sql   # сид (опционально)
```

### 3. Переменные окружения

Скопируйте пример и заполните значения:

```bash
cp .env.example .env.local
```

| Переменная | Назначение |
| --- | --- |
| `SUPABASE_URL` | URL проекта Supabase. |
| `SUPABASE_SERVICE_ROLE_KEY` | Секретный `service_role` ключ (только сервер). |
| `BUSINESS_TIMEZONE` | Зона, в которой заданы рабочие часы (по умолчанию `Europe/Moscow`). |
| `CRON_SECRET` | Секрет для защиты эндпойнта напоминаний. |
| `TELEGRAM_BOT_TOKEN` | Токен бота (опционально). |
| `RESEND_API_KEY` / `EMAIL_FROM` | Email-рассылка (опционально). |

> Если каналы уведомлений не настроены, напоминания не теряются: они логируются
> в консоль и помечаются отправленными — удобно для локальной разработки.

### 4. Запуск

```bash
npm run dev
```

Откройте http://localhost:3000. Админка — http://localhost:3000/admin.

## Проверка защиты от двойной брони

Нагрузочный тест шлёт 20 параллельных запросов на **один и тот же слот** и
проверяет, что прошёл ровно один (`201`), а остальные получили `409`:

```bash
npm run dev          # в одном терминале
npm run loadtest     # в другом (нужны применённые миграции и хотя бы одна услуга)
```

Можно указать другой адрес: `BASE_URL=https://my-app.vercel.app npm run loadtest`.

## Напоминания

Триггер `schedule_booking_notifications` при создании брони ставит в очередь
напоминание на время `starts_at − 2 часа`. Канал выбирается по контакту клиента:
адрес с `@` → email, иначе → Telegram.

Эндпойнт `POST /api/cron/send-reminders` забирает «созревшие» напоминания и
отправляет их. Он защищён заголовком `Authorization: Bearer <CRON_SECRET>`.

### Планировщик

**Vercel Cron (по умолчанию).** Файл [`vercel.json`](./vercel.json) запускает
эндпойнт каждую минуту. Vercel автоматически подставляет `CRON_SECRET` в
заголовок, если переменная задана в настройках проекта.

> На тарифе Hobby Vercel Cron ограничен запуском раз в день. Для запуска раз в
> минуту нужен тариф Pro — либо используйте `pg_cron` (см. `0003_cron.sql`).

**pg_cron (альтернатива).** Раскомментируйте и настройте `0003_cron.sql`, чтобы
Postgres сам дёргал эндпойнт через `pg_net` по расписанию.

## Деплой на Vercel

1. Запушьте репозиторий и импортируйте его в Vercel.
2. Добавьте переменные окружения из таблицы выше в настройках проекта.
3. Деплой. `vercel.json` автоматически подключит cron.

## Структура проекта

```
src/
  app/
    page.tsx                     список услуг
    book/[serviceId]/page.tsx    страница записи
    admin/                       админка + server actions
    api/
      services/                  GET услуги
      slots/                     GET доступные слоты
      bookings/                  POST бронирование (409 при конфликте)
      cron/send-reminders/       POST рассылка напоминаний
  components/
    Calendar.tsx                 месячный календарь
    BookingFlow.tsx              шаги записи (дата → слот → форма)
  lib/
    supabase/server.ts           админ-клиент (service_role)
    env.ts, time.ts, types.ts, notifications.ts
supabase/
  migrations/                    0001 схема, 0002 функции, 0003 cron (опц.)
  seed.sql                       тестовые данные
scripts/
  load-test.mjs                  нагрузочный тест
```
